## 🚀 Déploiement

Le portfolio est publié sur **GitHub Pages** à l’adresse suivante :  
https://seydinalimamoulayeyade.github.io/fullStack_portfolio/

Le projet suit une logique simple :
- le code source est versionné sur GitHub
- les changements sont poussés sur la branche principale
- le site est servi via GitHub Pages
- un pipeline GitHub Actions vérifie la qualité minimale du projet à chaque push et pull request

---

## 🔁 Pipeline CI/CD

Un workflow GitHub Actions est utilisé pour automatiser les contrôles de base :
- vérification de la présence des fichiers clés du site
- validation syntaxique des modules JavaScript
- contrôle des références de navigation principales

Objectif : rendre le dépôt plus fiable, détecter rapidement les erreurs simples, et exposer une première chaîne d’automatisation cohérente avec un positionnement Cloud & DevOps.

## ▶️ Lancer le projet en local

Comme il s’agit d’un site statique, il suffit d’ouvrir `index.html` dans un navigateur.

Pour un rendu plus propre en développement, tu peux aussi utiliser un serveur local, par exemple :

```bash
python3 -m http.server 8000
```

Puis ouvrir :

```text
http://localhost:8000
```
