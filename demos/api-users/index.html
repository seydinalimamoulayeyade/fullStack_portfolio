UserAPI

v1.2.0 Opérationnel

API RESTful de gestion des utilisateurs avec authentification JWT, rôles et permissions. Construite avec Node.js et Express.

Base URL https://api.userapi.dev/v1

Authentification

Toutes les requêtes (sauf POST /login) nécessitent un token JWT dans le header Authorization.

Authorization header
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
GET /users Récupérer tous les utilisateurs

Retourne la liste paginée de tous les utilisateurs. Requiert le rôle admin.

Paramètres de requête

page integer Page à afficher (défaut : 1)
limit integer Nombre de résultats par page (défaut : 20, max : 100)
role string Filtrer par rôle : admin | user

Réponse 200 OK 

{
  "success": true,
  "data": [
    {
      "_id": "64f2b3c8e1d9a7b23c4d5e6f",
      "name": "Seydina Limamou Laye Yade",
      "email": "seydina@example.com",
      "role": "admin",
      "createdAt": "2025-01-12T10:30:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 247
  }
}
POST /users Créer un utilisateur

Crée un nouvel utilisateur. Le mot de passe est haché avec bcrypt avant stockage.

Body (JSON)

name string* Nom complet (2–100 caractères)
email string* Email valide et unique
password string* Min 8 caractères, 1 majuscule, 1 chiffre
role string Défaut : "user"

201 Created 

{ "success": true, "data": { "_id": "..." } }

400 Bad Request 

{ "success": false, "error": "Email déjà utilisé" }
DELETE /users/:id Supprimer un utilisateur

Suppression définitive. Requiert le rôle admin. Un utilisateur ne peut pas se supprimer lui-même.

Cette action est irréversible. Les données de l'utilisateur sont supprimées définitivement de la base MongoDB.
